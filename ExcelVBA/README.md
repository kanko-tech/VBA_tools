# ExcelVBA

`ExcelVBA` ディレクトリには、Excel VBA で使うクラスモジュールを用途ごとに整理して配置しています。  
現在は、一次元配列、二次元配列、列名付きテーブルを扱うための 3 つのモジュール群があります。

## ディレクトリ構成

### `Vector`

一次元配列を扱うクラスモジュールです。

- 1 列または 1 行の `Range` を読み込む
- 一次元配列を保持する
- 要素参照、型変換、縦横出力を行う

向いている用途:

- 1 列だけをまとめて処理したいとき
- 型変換済みの値配列を作りたいとき
- 将来的な `Table` の列操作の基礎部品として使いたいとき

### `Matrix`

二次元配列を扱う基本クラスモジュールです。

- 矩形 `Range` を 2 次元配列として読み込む
- 行数・列数・セル値を参照する
- 行フィルタ、列選択、列更新を行う
- シートへまとめて出力する

向いている用途:

- 列名は不要だが、表データ全体を配列として扱いたいとき
- `Table` より一段低いレベルで表操作を組み立てたいとき

### `Table`

列名を持つ表データを扱うクラスモジュールです。  
内部的には `Matrix` を使い、列名ベースでの操作を提供します。

- ヘッダー付き `Range` を表として読み込む
- 列名で列を取得する
- 条件に合う行を抽出する
- 条件に合う行の特定列を書き換える
- 列名ベースで列を選択する

向いている用途:

- `pandas.DataFrame` に近い感覚で表を扱いたいとき
- 「ある列の条件に一致する行だけ、別の列を更新する」といった処理を書きたいとき

## 使い分けの目安

- 単一列の処理なら `Vector`
- 列名なしの表データなら `Matrix`
- 列名付きの実務データや条件抽出なら `Table`

## 今後の拡張イメージ

- `Vector` は列演算や条件ベクトル生成の強化
- `Matrix` は表データの基盤機能の拡張
- `Table` は `pandas` を参考にした高水準なテーブル操作の拡張
