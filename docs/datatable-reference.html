DataTable

列名付き表データモジュールの API 詳細

DataTable の公開 API を、列名ベースの読込・抽出・更新・出力の流れで整理しています。 クラス図、プロパティ、メソッドの前提条件、注意点、ユースケースを折りたたみ式で確認できます。

Properties

プロパティと参照系

Propertyis_loaded Property Get is_loaded() As Boolean読込済みかを返す

表データが読み込まれているかを返します。

入力なし
出力Boolean
前提なし
注意列名の妥当性までは再検証しません。
ユースケース初期化忘れを防ぐガードを書く。
Propertyrow_count Property Get row_count() As Long本文行数を返す

ヘッダーを除いた本文の行数を返します。

入力なし
出力Long
前提読込済みであること。
注意ヘッダー行はカウントしません。
ユースケース絞り込み後の件数確認。
Propertycol_count Property Get col_count() As Long列数を返す

保持している列数を返します。

入力なし
出力Long
前提読込済みであること。
注意列追加や選択後の構造確認に向いています。
ユースケース必要列がそろっているかを確認する。
Propertycolumn_names Property Get column_names() As Variant列名配列を返す

現在の列名一覧を一次元配列で返します。

入力なし
出力列名の一次元配列
前提読込済みであること。
注意返り値を編集しても元の列名は変わりません。
ユースケースUI 表示や外部出力で列一覧を確認する。
Propertymatrix Property Get matrix() As Matrix内部表現を Matrix で返す

本文データだけを Matrix として返します。

入力なし
出力Matrix
前提読込済みであること。
注意ヘッダー行は含みません。
ユースケース低レベルの行列処理へ戻したいとき。
Methodcol Function col(ByVal columnName As String) As Variant指定列を配列で返す

列名を指定して本文列を一次元配列で返します。

入力columnName As String
出力列データの一次元配列
前提読込済みで、列名が存在すること。
注意行数 0 でも列構造があれば返せる前提ではありません。内部的に Matrix 側の制約を受けます。
ユースケース特定列だけを別ロジックで処理する。
Methodcol_vector Function col_vector(ByVal columnName As String) As Vector指定列を Vector で返す

列を Vector 化して返し、条件判定や集計へつなげやすくします。

入力columnName As String
出力Vector
前提列名が存在し、依存クラス Vector が使えること。
注意列データを読込済みの新しい Vector として返します。
ユースケース金額列だけを取り出して sum や mean を使う。

Loading

読込系

Methodread_range Sub read_range(ByVal rng As Range, Optional ByVal hasHeader As Boolean = True)Range から表を読む

シート上の範囲をヘッダー付き表として読み込みます。

入力rng As Range, hasHeader As Boolean
出力なし。内部状態を更新。
前提rng が Nothing でないこと。
注意hasHeader:=False の場合は列名が col1, col2 ... になります。
ユースケースExcel 上の一覧表をそのまま DataTable 化する。
Methodread_matrix Sub read_matrix(ByVal src As Matrix, ByVal columnNames As Variant)Matrix と列名配列から読む

本文データと列名配列を別々に持っている場合の入口です。

入力src As Matrix, columnNames の一次元配列
出力なし。内部状態を更新。
前提列名数と src.col_count が一致すること。
注意列名の重複や空文字は内部初期化時に例外になります。
ユースケースMatrix で作った中間表に列名を付けて実務処理へ渡す。

Selection

抽出・選択系

Methodfilter_by_mask Function filter_by_mask(ByVal mask As Variant) As DataTableマスクで行抽出

ブール配列に応じて行を残した新しい DataTable を返します。

入力mask の一次元配列
出力DataTable
前提読込済みで、行数がある場合は mask 長が行数と一致すること。
注意0 件結果では列構造を保った空 DataTable を返します。
ユースケースVector 由来の条件配列で表全体を絞り込む。
Methodfilter_by_equals Function filter_by_equals(ByVal columnName As String, ByVal matchValue As Variant) As DataTable一致条件で抽出

指定列が一致する行だけを返します。

入力columnName, matchValue
出力DataTable
前提列名が存在すること。
注意比較不能な値は一致しません。
ユースケースstatus = "OK" の行だけ取り出す。
Methodfilter_by_in Function filter_by_in(ByVal columnName As String, ByVal matchValues As Variant) As DataTable候補群のいずれかで抽出

複数候補のいずれかに一致する行だけを返します。

入力columnName, matchValues の一次元配列
出力DataTable
前提列名が存在し、候補配列が一次元であること。
注意候補数が多いほど比較回数は増えます。
ユースケース複数部署や複数区分に該当する行をまとめて抜く。
Methodfilter_by_contains Function filter_by_contains(ByVal columnName As String, ByVal searchText As String, Optional ByVal caseSensitive As Boolean = False) As DataTable部分一致で抽出

指定列の文字列表現に検索文字列を含む行だけを返します。

入力columnName, searchText, caseSensitive
出力DataTable
前提列名が存在すること。
注意Error や Empty はヒットしません。
ユースケース名称列からキーワード検索を行う。
Methodfilter_by_all_equals Function filter_by_all_equals(ByVal columnNames As Variant, ByVal matchValues As Variant) As DataTable複数列の AND 条件

複数列の条件をすべて満たす行だけを返します。

入力columnNames, matchValues の一次元配列
出力DataTable
前提両配列が一次元で、長さが一致すること。
注意列ごとに順番対応で評価されます。
ユースケース部署と年度の両方が一致する行だけを集める。
Methodfilter_by_any_equals Function filter_by_any_equals(ByVal columnNames As Variant, ByVal matchValues As Variant) As DataTable複数列の OR 条件

複数列条件のいずれかを満たす行を返します。

入力columnNames, matchValues の一次元配列
出力DataTable
前提両配列の長さが一致すること。
注意条件数が多い場合でも短絡評価はしません。
ユースケース複数列のうちどれかが特定値なら抽出する。
Methodselect_columns Function select_columns(ByVal columnNames As Variant) As DataTable必要列だけ残す

指定した列名だけで新しい DataTable を作ります。

入力columnNames の一次元配列
出力DataTable
前提すべての列名が存在すること。
注意列順は配列で渡した順に従います。
ユースケース公開用出力に必要な列だけを残す。

Mutation

更新・整形系

Methodadd_column Sub add_column(ByVal columnName As String, Optional ByVal values As Variant)列を追加する

末尾に新しい列を追加します。値省略時は空欄列として追加されます。

入力columnName, 任意の values
出力なし。内部状態を更新。
前提列名が空でなく未使用であること。
注意行数がある場合、値配列の長さは行数と一致している必要があります。
ユースケース判定結果列や補助列を追加する。
Methodrename_column Sub rename_column(ByVal oldName As String, ByVal newName As String)列名を変更する

既存列の名前を付け替えます。

入力oldName, newName
出力なし。内部状態を更新。
前提oldName が存在し、newName が空や重複でないこと。
注意大文字小文字違いだけなら同一列扱いです。
ユースケース外部データの列見出しを内部命名へそろえる。
Methodsort_by Sub sort_by(ByVal columnName As String, Optional ByVal ascending As Boolean = True)指定列で並べ替える

指定列の値を基準に本文行を並べ替えます。

入力columnName, ascending
出力なし。内部状態を更新。
前提列名が存在すること。
注意比較不能な組み合わせは並べ替え判断で False 扱いです。
ユースケース金額列や日付列で一覧を昇順・降順に整える。
Methodset_by_mask Sub set_by_mask(ByVal mask As Variant, ByVal columnName As String, ByVal newValue As Variant)条件行だけ更新する

マスクが True の行だけ、対象列を新しい値へ更新します。

入力mask, columnName, newValue
出力なし。内部状態を更新。
前提mask が一次元で、長さが行数と一致すること。
注意0 件テーブルでは何もせず終了します。
ユースケース抽出済みフラグ配列を使って一括で値を書き換える。
Methodset_by_equals Sub set_by_equals(ByVal conditionColumnName As String, ByVal matchValue As Variant, ByVal targetColumnName As String, ByVal newValue As Variant)一致条件で更新する

条件列が一致した行だけ、対象列を書き換えます。

入力条件列名、条件値、更新列名、更新値
出力なし。内部状態を更新。
前提両列名が存在すること。
注意内部的には一致マスクを作って set_by_mask を呼びます。
ユースケースstatus = "NG" の行だけフラグ列を更新する。
Methodset_column Sub set_column(ByVal columnName As String, ByVal values As Variant)列全体を置換する

指定列の値を新しい配列へ丸ごと置き換えます。

入力columnName, values
出力なし。内部状態を更新。
前提列名が存在し、値配列長が行数と一致すること。
注意検証は Matrix 側の set_column_values に準じます。
ユースケース再計算済みの列全体を一括で差し替える。

Output

出力系

Methodto_range Sub to_range(ByVal topLeft As Range, Optional ByVal includeHeader As Boolean = True)シートへ書き戻す

現在の表をシートへ出力します。必要ならヘッダーも含めます。

入力topLeft As Range, includeHeader As Boolean
出力なし。シートへ書込。
前提読込済みで topLeft が有効であること。
注意本文 0 件でも includeHeader:=True ならヘッダーのみ出力します。
ユースケース加工結果を別セル位置へ一覧として書き戻す。

State

状態管理系

Methodclear Sub clear()内部状態を初期化する

本文、列名、列名マップ、読込状態をまとめて初期化します。

入力なし
出力なし。内部状態を初期化。
前提なし
注意再利用前に必要な列構造も消えるため、再度読込が必要です。
ユースケース同じ変数で別テーブルを処理する前に完全リセットする。
